# Presentation Story Builder

## Purpose
Build the capstone narrative so it lands: anchored by a walkthrough of the agentic workspace and a live dashboard demo, with the opportunity, deployment week, before/after evaluation, and next steps around them. Every team member presents a part.

## When to use
- After the dashboard and evaluation are mostly complete, in presentation prep.

## Inputs to read
All of `deliverables/`, `governance/governance.md`, the deployment log, evaluation results, `CLAUDE.md`, `knowledge/managing-intelligence/managing-intelligence.md` (capstone guidance).

## Story arc (organize the talk around this)
1. The classroom problem we identified.
2. The user and the decision it supports.
3. Why Claude is useful here.
4. Why governance is necessary.
5. Dashboard / app demo (live — the point of the talk).
6. Agentic workspace walkthrough (`CLAUDE.md`, data connection, skills, subagents, workflows — why each earns its place).
7. The AI Council process.
8. Evaluation results (baseline → revised, before-and-after).
9. What changed after testing and review.
10. Remaining risks.
11. What we would build next.

## Procedure
1. Draft a beat-by-beat outline mapping each beat to a speaker (every member presents).
2. For the demo, script the exact path (including the regression-oracle question) and prepare a screenshot/recording backup.
3. Tie the governance and evaluation beats to concrete evidence (Decision Log, before/after numbers, a salient episode).
4. Time the talk to ~20 minutes; rehearse; mark where Q&A is likely.
5. End on a defensible answer to "Why was this worth building?"

## Guardrails
- The live demo is the anchor — don't replace it with slides.
- Avoid AI hype; state exactly what Claude helps the user see, decide, or do.
- Pair with the `professor-defense-prep` skill before the talk.
